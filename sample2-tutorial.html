ACT-R Tutorial: Declarative Memory Module- Probabilistic Retrieval Depends On Activation Levels

Author: Athirah Nasrullah
Date: September 2025
Purpose: Technical onboarding guide for modeling human memory in ACT-R.

1. Introduction

In the ACT-R cognitive architecture, declarative memory stores factual knowledge as structured units called chunks. Unlike simple databases, ACT-R models human memory with psychological fidelity: retrieval is probabilistic, influenced by activation levels, and subject to noise and decay.

This tutorial explains how declarative memory works in ACT-R, focusing on:

  • The structure of chunks and declarative memory.
  • How activation determines retrieval success and latency.
  • Simulating probabilistic retrieval in a simple arithmetic task.

2. Chunks and Declarative Memory

Each piece of factual knowledge is represented as a chunk, which instantiates a predefined chunk type. Chunks reside in long-term declarative memory and are accessed via the retrieval buffer.

Example: Arithmetic Facts

(chunk-type arithmetic-fact
  problem
  answer)

(chunk add-2-3
  isa arithmetic-fact
  problem (2 + 3)
  answer 5)

(chunk add-4-1
  isa arithmetic-fact
  problem (4 + 1)
  answer 5)

These chunks can be added to memory using add-dm:

(add-dm 
  add-2-3 
  add-4-1)

3. Activation and Retrieval

When a model attempts to retrieve a chunk, ACT-R computes an activation value for each matching chunk. Retrieval is probabilistic: the chunk with the highest activation is most likely to be retrieved, but noise and competition can lead to errors or delays.

3.1 Base-Level Activation

Each chunk has a base-level activation (B_i) that reflects its history of use. It increases with frequency and recency of exposure and decays over time. The formula is:

B_i = ln(Σ t_j^{-d}) + B
where:
• t_j = time since j-th use of the chunk
• d = decay parameter (typically ~0.5)
• B = asymptotic value offset
A fact like "2 + 3 = 5" that has been used frequently and recently will have higher base-level activation than a rarely used fact.

3.2 Activation from Context (Associative Activation)

Chunks also receive associative activation based on how well their content matches the current context (i.e., other chunks in buffers). This is computed as:

A_i = B_i + Σ W_j * e^{-d * |S_j - S_i|}
where:
• W_j = weight of source chunk j
• S_j, S_i = spreading activation values
• d = decay parameter

In practice, ACT-R automatically computes this when you specify retrieval requests with partial or full pattern matching.

3.3 Retrieval Threshold and Noise

The total activation A_i determines both whether a chunk is retrieved and how quickly:

  • If A_i < threshold, retrieval fails.
  • Response time is modeled as: RT = F * exp(-A_i)
  • Stochastic noise is added to activation during retrieval.
Key Parameters:
• :rt – Base time for retrieval (default ~0.05 sec)
• :ans – Noise parameter (default ~0.25)
• :lf – Learning rate (controls decay)
• :mp – Maximum partial matching bonus

4. Simulating Probabilistic Retrieval

Let’s build a model that retrieves arithmetic facts. We’ll simulate how activation affects retrieval success over time.

Complete Model Code

; Load ACT-R
(load "act-r/load-act-r.lisp")

; Define chunk type
(chunk-type arithmetic-fact problem answer)

; Add facts to declarative memory
(add-dm 
  (fact-2+3 isa arithmetic-fact problem (2 + 3) answer 5)
  (fact-4+1 isa arithmetic-fact problem (4 + 1) answer 5)
  (fact-7-2 isa arithmetic-fact problem (7 - 2) answer 5))

; Set retrieval threshold to make failures possible
(set-parameter-value :retrieval-threshold 0.5)

; Goal buffer: current problem to solve
(chunk-type goal problem status)

; Initial goal
(goal-focus (isa goal problem (2 + 3) status solve))

; Production: Start retrieval
(P retrieve-answer
   =goal>
     isa goal
     problem =prob
     status solve
   ==>
   =goal>
     status retrieving
   +retrieval>
     isa arithmetic-fact
     problem =prob)

; Production: Successful retrieval
(P succeed
   =goal>
     isa goal
     status retrieving
   =retrieval>
     isa arithmetic-fact
     answer =ans
   ==>
   =goal>
     status done
   !output! ("Retrieved answer: " =ans))

; Production: Failed retrieval
(P fail
   =goal>
     isa goal
     status retrieving
   =retrieval>
     error
   ==>
   =goal>
     status failed
   !output! ("Retrieval failed. Try again later."))

; Run the model for up to 5 seconds
(run 5)

Expected Behavior

Run 1: "Retrieved answer: 5" (fast, if activation is high)
Run 2: "Retrieval failed..." (if activation decays or noise interferes)
Variability: Response time and success rate will vary across runs due to subsymbolic noise.

5. Observing Activation in Real Time

You can inspect a chunk’s current activation using:

(print-activation fact-2+3)

To monitor retrieval attempts:

(trace-on retrieval)

This will show in the console whether retrieval succeeded, failed, or matched partially.

6. Practical Tips for Modeling

  • Calibrate parameters using empirical data (e.g., human response times).
  • Use partial matching to simulate retrieval from incomplete cues.
  • Model learning by presenting facts multiple times—activation increases with practice.
  • Validate your model by comparing retrieval accuracy and latency to human data.

7. Conclusion

ACT-R’s declarative memory system provides a psychologically plausible model of human memory retrieval. By combining base-level and associative activation with noise and thresholds, it captures key phenomena such as:

  • Forgetting over time
  • Primacy and recency effects
  • Interference between similar memories
  • Speed-accuracy tradeoffs

Mastery of activation dynamics is essential for building realistic cognitive models in AI research, education, and human-computer interaction.
